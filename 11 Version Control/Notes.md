## Version Control

### 4.18.2

1st part => 4
2nd part => 18
3rd part => 2

3rd Part (Last Part) - Minor Fixes (Optional)

2nd Part - Recommended Bug Fix (Security Fixes) or new feature (recommended)

1st Part - Major Release / Breaking Update
Its not recommended to change it in exsiting code, it may cause code break (major release)
We can use latest version if we are writing code from scratch

^ (Carrot Symbol) means Compatible with version, keep the major release same, recommended and minor fixes can be change.
^4.18.2 => 4.18.2 to 5.0.0 (5.0.0 not included)

~ means approximately equivalent version, only minor fixes are allowed to change or update.
